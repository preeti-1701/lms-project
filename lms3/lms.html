Security Requirements

Secure by Design

All communication is HTTPS-encrypted. User data is protected at every layer.

🔒

HTTPS / TLS

All traffic encrypted in transit. No plain-text data ever sent over the wire.

🔑

Password Hashing

Passwords stored as bcrypt hashes. No plaintext credentials in the database.

🎫

JWT Sessions

Stateless authentication tokens with expiry — no session fixation attacks.

🛡️

Role-Based Access

Students, instructors, and admins each have isolated permissions and routes.

🚫

Input Validation

Server-side sanitization prevents SQL injection and XSS attacks.

📋

Audit Logs

All admin actions are logged with timestamps for full accountability.
